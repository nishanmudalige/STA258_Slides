Why Learn R?

Every statistician on Earth uses code.
You’re about to learn why.

Three reasons:

  • Reproducibility — every step is written down. Re-run it next year and get the same answer.
  • Scale — the same script that analyzes 10 rows also analyzes 10 million.
  • Sharing — send your code, not screenshots of your clicks.

2.1 The Statistical Computing Language

Definition — R
R is a programming language designed for data manipulation, statistics, graphics, and reproducible analysis.

R is good at:

  • arithmetic on numbers, vectors, matrices
  • producing publication-quality graphs
  • using built-in and user-contributed packages
  • saving entire analyses as scripts that anyone can re-run

Emphasize that R is free, open-source, and the standard tool in statistics.

Point-and-click vs Code

Point-and-click software

Quick at first. But every click is invisible — hard to repeat, hard to audit, hard to share.

R scripts

Every step is written down. The script is the analysis — you can re-run it, version it, share it, or modify one line and re-run everything.

Bottom line: R makes statistical work reproducible, transparent, and automatic.

2.2 R vs RStudio

R
The programming language. The thing that does the calculations.

RStudio
The interface. Where you write code, see plots, view files, and read output.

Analogy: R is the engine. RStudio is the dashboard.
You can drive an engine without a dashboard — but you wouldn’t want to.

Install order matters: install R first, then RStudio on top.

2.2.1 Installing R

Go to cran.r-project.org and pick your operating system.

  • Windows → Download R for Windows → base → run the installer
  • macOS → Download R for macOS → pick the matching .pkg file
  • Linux → follow your distribution’s instructions

Once installed, R can run from a terminal — but in this course we always use it through RStudio.

2.2.2 Installing RStudio

RStudio
An integrated development environment (IDE) designed for working with R.

RStudio gives you:

  • a script editor (write code)
  • a console (run code)
  • environment & history (see your objects)
  • files / plots / packages / help (everything else)

Mention: RStudio is now made by Posit. Same product, new company name.

The Four RStudio Panes

Pane What it does
Source Write and edit .R scripts and .qmd files
Console Run commands directly and see output
Environment / History Inspect saved objects and past commands
Files / Plots / Packages / Help Browse files, view graphs, manage packages

Shortcut: Cmd + Enter (Mac) or Ctrl + Enter (Windows) runs the current line.

A Typical Workflow

  1. Open RStudio
  2. Create a new R script (File → New File → R Script)
  3. Type your code in the Source pane
  4. Hit Cmd/Ctrl + Enter to run the current line
  5. See results in the Console or Plots pane
  6. Save the script so you can re-run later

Saving the script is the whole point — your analysis lives in the file, not in the console.
